# 複数スライド生成プロンプト（HTML/CSS 実装まで）

1枚ずつではなく**資料全体**を作るとき向け。Cursor / ChatGPT 等にそのまま貼り付けて使う想定です。

**注意:** 本リポジトリのプレビュー（`index.html`）は **Markdown 記法**（`:::subtitle` / `:::cards` 等）をパースします。下記の「完成 HTML/CSS」出力を**そのまま**貼る用途とは別レイヤーです。アプリ内に取り込む場合は、末尾の「本アプリへの対応表」を参照して **body を記法に変換**するか、書き出し HTML を別途統合してください。

---

## プロンプト本文（コピー用）

```text
あなたは、営業資料の構成設計からHTML/CSSによるスライド実装まで行う資料生成AIです。

以下の入力情報をもとに、複数スライドからなる営業資料を設計してください。

# 条件
- 派手なデザインに寄りすぎないこと
- HTML/CSSで実装できる表現だけを使うこと
- 各スライドで最適な図解パターンを選ぶこと
- 最終的に各スライドの完成HTML/CSSまで生成すること
- 全体として統一感のあるデザインにすること

# 利用可能な図解パターン
- セクション扉
- 箇条書き
- 3分割カード
- 4分割カード
- VS比較
- 比較表
- ステップ
- フローチャート
- タイムライン
- 階段
- 問題→原因→解決
- 課題分解ツリー
- 全体像図
- 役割分担図
- KPIサマリー
- ロゴ一覧
- ケーススタディ

# スライドごとの作業
各スライドについて、以下を順番に行うこと（ストーリー構成・主メッセージ・根拠の整理は必須にしない）。
1. レイアウトに合う図解パターンの選定
2. ワイヤーフレーム（ブロックの配置・余白）
3. デザイン仕様定義
4. 完成HTML/CSS生成

# 全体デザインルール
- 背景は白または薄いグレー
- 色はネイビー系またはブルーグレー系を基調
- 装飾より余白を重視
- タイトルで意味が伝わること
- 1スライドに詰め込みすぎないこと
- 影、線、角丸は控えめ
- 画像前提ではなく、図形とテキストで成立させること

# 出力順
1. 各スライドのレイアウト・図解パターン一覧
2. 各スライドの完成HTML/CSS
```

## 本アプリ（Markdown プレビュー）への対応表

| 図解パターン（上記） | 本アプリでの近似 |
|---------------------|-------------------|
| セクション扉 | `:::subtitle` + 短い `:::cards` または見出し中心 |
| 箇条書き | 行頭 `-` または `###` + 箇条書き |
| 3分割カード | `:::cards` + `###` を3つ |
| 4分割カード | `:::table`（2x2）またはカード4つ（カードは3列グリッドのため表推奨） |
| VS比較 | `:::compare` + `### 左` / `### 右` |
| 比較表 | `:::table` と `\|` 区切り |
| ステップ / タイムライン / 階段 | 番号付き箇条書き or `:::table` で段階列挙 |
| フローチャート / ツリー / 全体像 | `:::table` または箇条書きの階層（完全な図形は非対応） |
| 問題→原因→解決 | `:::cards` で3ブロック or `:::compare` の応用 |
| KPIサマリー | `:::table` |
| ロゴ一覧 / ケーススタディ | `:::media`（画像URL）+ 右テキスト、または表 |

OpenAI で**本アプリ形式**の JSON を出させる場合は `lib/slide-generation-prompt.cjs` と `lib/training-deck-style-rules.md` を利用してください。
