# 研修ご案内資料スタイル（生成AI研修・DX 系の参照ルール）

このファイルは「研修内容のご案内」系 PDF の**ボリューム感・配置の考え方・書き方**から抽出したルールです。スライド本文生成時に従う。

## トーンと対象読者

- **文系・非 IT・非エンジニア**でも追える説明粒度。専門用語は必要最小限にし、入れた場合は短い言い換えを添える。
- **実務直結**を常に明示する（「現場で使える」「学んだその日から」「業務を想定した」など具体シーン）。
- **短期間で習得・即実践**のベネフィットをサブメッセージやカードの一行目で示す。
- 上から目線にならず、「初学者でも」「挫折せず」など**安心感**と**前向きな成果**のバランス。

## 情報のボリューム（1 枚あたり）

- **1 スライド = 1 メッセージ**が基本。詰め込みすぎない。
- 本文は**長い段落を書かない**。意味のかたまりは **`:::cards`（3 列）**、**`:::table`**、**`:::compare`（左/右）** に落とす。
- 箇条書きは **1 行は短め**、項目は **3〜5 個程度**が目安（一覧表は表に任せる）。

## 構成・章立てのニュアンス

- **大見出し**は「〇〇について」だけより、**メッセージが伝わる短い結論型**を優先（例:「なぜ今、生成 AI 研修が必要なのか？」）。
- 章・パートは **番号 + 英語サブキャッチ**の**雰囲気**を取り入れてよい（例: title に「01」「カリキュラム」、subtitle に Curriculum Overview 系の一言英語は**短ければ可**。無理に英語にしない）。
- **三部構成**（例: 基礎 → 思考力・応用 → DX・実務）のように**段階が伝わる並べ方**を好む。
- **特徴・メリット**は **「3 つの特徴」**のように **3 分割**（カード 3 枚）で示す。

## 課題・共感パート

- 企業現場の**リアルな課題**は、**短い引用風**（「」）または **左列: 課題 / 右列: 方向性** の `:::compare` で示す。
- **属人化・手作業・現場がついていない**など、DX 文脈でよく使う対比軸を活用してよい。

## 中身の型（研修・ご案内でよく使う）

- **表紙相当**: キャッチに近い title + `:::subtitle` で副題（例: `-実務で使える〜-` のようなトーン）+ `:::table` で日付・対象・種別など。
- **プログラム概要**: 3 部の役割は **`:::cards`** で「第一部 / 第二部 / 第三部」または「学べること」の 3 ブロック。
- **カリキュラム詳細**: モジュール名は `###` 見出し + 箇条書き。**実践演習**は明記（【実践演習】〇〇）。
- **ターゲット・レベル**: ペルソナや **レベル 0〜10** のような**段階表**は `:::table` が適する。
- **効果・費用対効果**: Before/After、**作業時間の数値**などは `:::compare` または表。

## ビジュアル・デザインの前提（本文から示せる範囲）

- 「**画像やイラストを用いて分かりやすい**」前提で、**本文は図解・図表のキャプション**に近い短さにする。
- スライドにそのまま載せる本文なので、**装飾・配色の指示文は書かない**（レイアウト記法のみ）。

## 書かないこと・深入りしないこと（本資料の範囲に合わせる）

- **モデル開発・機械学習アルゴリズム・Python 実装・MLOps・統計解析・研究開発向け**の深入りはしない（「対象外」として触れる程度にとどめる）。
- 研修の**助成金・スケジュール**など、ブリーフにない制度詳細を**捏造しない**（ブリーフに近いトーンのみ）。

## 記法との対応（必須）

- 各スライドの body **先頭**に `:::subtitle` … `:::`（**1〜2 行のキャッチ**）。
- 続けて **`:::cards` / `:::table` / `:::compare` / `:::media`** のいずれかで構造化。
- `:::compare` は **`### 左`** と **`### 右`** のみ（2 列）。
