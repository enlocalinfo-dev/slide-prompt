# Cursor 上でのスライド文章・HTML 生成

Web アプリの LLM 連携は使わず、**このフォルダのブリーフを Cursor に読ませて** HTML/CSS と具体文を書いてもらう想定です。

## 手順（最短）

1. `brief.template.md` をコピーして内容を埋める（複数ファイルに分けてもよい）。
2. 新しいチャットで `@brief.md`（あなたが保存した名前）を添付。
3. `cursor-生成プロンプト.md` の「依頼文」ブロックをコピーして貼る。
4. 生成された `output/deck.html` をブラウザで開いて確認。

## 「バーっとアップロード」について

- Cursor では **チャットに `@` でファイルを複数指定**できます。ブリーフに加え、メモ・議事録・たたき台の txt / md をまとめて添付すると、文脈を踏まえた本文になりやすいです。
- フォルダごと扱いたい場合は、ルートに近い場所に md を置き、`@フォルダ名` や個別ファイルを複数指定してください。

## プレビューツール（任意）

リポジトリ直下の `index.html` は、**見出し＋本文の編集と 16:9 プレビュー**用です。Cursor で生成した本文をコピーして貼り付け、分量感を確認できます。

## 分量・トンマナ（参考）

法人向け提案PDFに近い **情報密度・文体** を意識する場合は `slide-tonmana-rules.md` を参照してください（100%固定のルールではありません）。今後インプットする文案・制約も、ここに沿って統合しやすくします。

## インプット参照（記事・長文）

スライドや Genspark プロンプト作成時の文脈として使うテキストを置けます。

- 例：`input-提案書_決裁と構成の参考記事.md`（決裁者視点・通る／通らない構成の記事ドラフト）

チャットでは `@batch/input-提案書_決裁と構成の参考記事.md` のように添付して使ってください。

## 20枚カリキュラム（HTML）

`output/curriculum-20h/` に一覧・各スライド・編集画面があります。原稿の再生成は `publish-slides-data.mjs` → `slides-data.json` 更新後、`generate-slides.mjs` で静的 HTML を出力します。
